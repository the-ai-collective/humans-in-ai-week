# Event Formats

Humans in AI Week has six day-specific event formats. Each format has its own agenda, best practices, and HQ Opening script. Pick the one that fits your community.

---

## Quick reference

| Day | Format | Duration | Ideal Size | Cost Level | Complexity | Best For |
|-----|--------|----------|-----------|------------|------------|----------|
| Monday | [Demo Night](./monday-demo-night/) | 2.5h | 50–150 | Low ($200–500) | Medium | Builder communities, tech-forward chapters |
| Tuesday | [Research Roundtable](./tuesday-research/) | 2.5h | 30–100 | Low ($200–500) | Medium | University towns, research communities |
| Wednesday | [Discussion Meetup](./wednesday-discussion/) | 2h | 15–60 | Lowest ($0–200) | Low | Any chapter, first-time hosts, intimate groups |
| Thursday | [Dinner](./thursday-dinner/) | 3h | 20–80 | Highest ($500–3,000) | High | Well-funded chapters, executive audiences |
| Friday | [Workshop](./friday-workshop/) | 3h | 20–60 | Low ($200–500) | High | Technical communities, hands-on learners |
| Saturday | [Hackathon](./saturday-hackathon/) | 8h | 30–100 | High ($800–3,000) | Highest | Large technical chapters, university partnerships |

---

## Decision matrix

### By chapter size

- **Small (under 30 members)**: Discussion Meetup or Dinner
- **Medium (30–80)**: Any format works. Demo Night and Workshop are crowd favorites
- **Large (80+)**: Demo Night, Hackathon, or Research Roundtable scale best

### By community type

- **Technical / builder**: Demo Night → Workshop → Hackathon
- **Academic / research**: Research Roundtable → Discussion Meetup
- **Business / professional**: Dinner → Discussion Meetup → Research Roundtable
- **General / mixed**: Discussion Meetup → Demo Night → Workshop
- **First-time hosts**: Discussion Meetup (lowest complexity, lowest cost)

### By budget

- **$0–200**: Discussion Meetup (minimal requirements)
- **$200–500**: Demo Night, Research Roundtable, Workshop
- **$500–3,000**: Dinner, Hackathon (require catering, full-day venue)

## What's in each format folder

Every format folder contains three files:

| File | What it covers |
|------|---------------|
| `agenda.md` | Minute-by-minute run of show, setup checklist, equipment list, required roles |
| `best-practices.md` | What makes it great, common mistakes, scaling guidance, topic track pairings |
| `hq-opening.md` | 5-minute opening script — non-negotiable, every HAIW event starts with this |

---

## Combining formats

You can run multiple events during the week. Popular combinations:

- **Wednesday Discussion + Saturday Hackathon** — Discussion surfaces problems, hackathon builds solutions
- **Monday Demo Night + Friday Workshop** — Demos inspire, workshop teaches the tools
- **Tuesday Research + Thursday Dinner** — Research informs, dinner deepens over food

---

## Can I pick a different day?

Yes. The day assignments are defaults, not requirements. If your venue is only available on Wednesday but you want to run a Demo Night — do it. The format matters more than the day. Just update your event listing and use the correct format files.

---

## Topic tracks are optional

Formats are the primary organizing unit. Topic tracks (in [`/content/discussion-guides/`](../content/discussion-guides/)) are optional overlays that add thematic depth. Each format's `best-practices.md` recommends which tracks pair well. See [`/content/README.md`](../content/README.md) for the full list of six tracks.
